# generate-creatives

把品牌資訊變成可投放的廣告素材。核心產物是
**`composition_prompt`：一段強結構化的自然語言，貼進 GPT（或其他生圖工具）就能生出廣告主視覺**。
依使用者狀況分兩種模式：**已有文案只補構圖**，或**連文案一起發想多組**。

> 重要前提（實測得到）：新版 GPT 生圖模型的**繁體中文渲染已很強**，文字直接寫進 prompt 即可，
> 幾乎不糊不錯字。

## 流程

### 1. 開場：先問「要做到哪一步」（決定模式）
**第一件事就主動問使用者**（這是分流關鍵，別跳過）：

> 「你已經有廣告文案了嗎？還是文案也要一起發想？」

- **模式 A — 已有文案，只補構圖**：使用者提供現成文案（標題／內文／CTA），
  你**只為它產 `composition_prompt`**（圖中文字＝照搬他的文案，不亂改）。可一則文案產多個視覺版本。
- **模式 B — 連文案一起發想**：你產 **N 組（預設 5）「文案 ＋ 構圖 prompt」**，衝量提高可用率。

### 2. 收 brief（用「一步一步詢問」收集，不要一次丟一張表單）
逐步問、每次只問一小組，並附上預設值讓使用者能直接說「用預設」或補充；答完一步再問下一步。建議順序：

1. **品牌 / 產品**：`brand_name`、`product` —— 必要，先問這個。
2. **賣點 / 優惠**：`usp[]`(賣點)、`offer`、`price`(可選)。
3. **調性 / 語言**：`brand_tone`(語氣)、`target_language`(預設 zh-TW)。
4. **合規 `compliance`**：該產業若有限制（禁用詞／必要揭露／年齡標示等）就問，沒有就留空。
5. **尺寸 `aspect`**：預設 `1:1`；可 `1:1` / `4:5` / `9:16`。
6. **有沒有額外要跟 Claude 說的話**：任何想交代的產出指示（風格偏好、必避免的元素、特定要求…），沒有就略過。這是**給 Claude 用來引導產出的指示，不寫進 `brief`、也不存進輸出 JSON**。

收齊後簡述一次 brief 給使用者確認，再往下產。

依模式再各補問：
- 若使用者選擇**模式 A**：請使用者**貼現成文案**（每則的 `headline` / `primary_text` / `cta`），以及**一則文案要幾個視覺版本**（`K`，預設 2）。
- 若使用者選擇**模式 B**：`N`(預設 5)、切角來源（二選一）：
  - **貼參考**：使用者貼競品廣告文案／連結／截圖 → 當場拆解切角與鉤子。
  - **純品牌 brief**：沒有參考時，依「品類 + brief」自行提 2～3 個切角給使用者挑。

確認後再產。

### 3. 產出通則（安全 / 合規 / 結構，A、B 都適用）

**安全 / 合規**
- 可使用客戶提供的素材（logo、產品照、過往視覺等）。
- 不要捏造 brief 沒給的價格／折扣／保證（模式 A 也不得改寫使用者文案的承諾）。
- 遵守 `compliance`。

**產出結構**（先知道目標形狀，兩模式都填這個）
```json
{ "id":"...", "created_at":"<UTC ISO>",
  "mode":"copy_only | full",
  "brief":{ "brand_name":"...","product":"...","offer":"...","price":null,
            "target_language":"zh-TW","brand_tone":"...","usp":["..."],
            "compliance":"...","aspect":"1:1" },
  "creatives":[
    { "angle":null, "hook_formula":null, "funnel_stage":null,
      "copy":{ "primary_text":"...","headline":"...","cta":"..." },
      "content":{ "title":"...","offer_badge":"...","cta":"..." },
      "composition_prompt":"純視覺：主題/主體/場景/配色/光線/氛圍/比例＋每段文字的位置；文字一律用 {{content.欄位}} 佔位引用＋合規負面表列" }
  ] }
```
- **`content` 與 `composition_prompt` 一定要切開**：`content` 放「圖中要出現的文字」（標題、優惠標、CTA…，依需要增減欄位），是可單獨調整的文字槽；`composition_prompt` **只描述視覺與文字位置**，文字一律寫成 `{{content.欄位}}`（如 `{{content.title}}`、`{{content.cta}}`），**不要把字串寫死在 prompt 裡** → 之後使用者改 `content` 的字就能直接改圖上的字，視覺描述完全不用動。
- `copy` = 投放後台的貼文文案；`content` = 畫在圖上的文字，兩者可不同。
- 模式 A：`mode="copy_only"`，`copy`/`content` 取自使用者提供的文案，`angle/hook_formula/funnel_stage` 可為 `null`；同一則文案的多個視覺版本各算一筆 `creatives`。
- 模式 B：`mode="full"`，各筆填齊 `angle/hook_formula/funnel_stage`。

**範例**（填好的樣子；改 `content` 的字，圖上的字就跟著換，視覺描述不用動）
```json
{
  "id": "20260604-153000-綠野保養",
  "created_at": "2026-06-04T07:30:00Z",
  "mode": "full",
  "brief": {
    "brand_name": "綠野保養", "product": "植萃保濕精華", "offer": "首購省 $100",
    "price": null, "target_language": "zh-TW", "brand_tone": "清新自然",
    "usp": ["100% 植萃", "30 天有感"], "compliance": "", "aspect": "1:1"
  },
  "creatives": [
    {
      "angle": "痛點放大", "hook_formula": "痛點放大", "funnel_stage": "MOFU",
      "copy": {
        "primary_text": "換季乾到脫皮？一滴植萃精華，30 天找回水潤光。🌿",
        "headline": "乾肌救星", "cta": "立即選購"
      },
      "content": {
        "title": "乾肌救星",
        "offer_badge": "首購省 $100",
        "cta": "立即選購"
      },
      "composition_prompt": "清新自然風格的保養品廣告，柔和晨光、淺綠與米白配色、水珠與綠葉點綴，1:1 比例。圖中文字：頂部大標 {{content.title}}、左上徽章 {{content.offer_badge}}、右下 CTA 按鈕 {{content.cta}}；位置清楚、留白充足、不雜亂。畫面不得出現：雜亂文字、其他品牌商標。"
    }
  ]
}
```

### 3A. 模式 A 產出（只補構圖）
為使用者的**每一則文案**產 `K` 個 `composition_prompt`，每個走**不同視覺方向**（場景／構圖／配色／氛圍要有差異，不是微調）。
- `content` 圖中文字**照搬使用者文案**（標題／CTA／優惠標…），不得擅自改寫或新增承諾；`composition_prompt` 用 `{{content.欄位}}` 引用它們、只描述視覺與位置。
跳過 §3B 的文案發想（模式 A 不需要）；`composition_prompt` 仍依 §3C 規格，再到 §4。

### 3B. 模式 B 產出 N 組（衝量 ＋ 真多樣性）
內建這套「結構骨架」當**推理依據**（不是查表，是用它逼出差異）：
- **hook 公式**：痛點放大 / 模式中斷 / 好奇缺口 / 社會證明 / 急迫稀缺 / 利益直球 / 故事代入
- **漏斗階段**：TOFU（認知）/ MOFU（考慮）/ BOFU（轉換）
- **情緒觸發**：FOMO、貪好康、歸屬感、爽快感、懷舊、勝負慾…

**多樣性鐵則**：N 組之間，各取**不同的（切角 × hook 公式 × 漏斗階段）組合**，
**禁止只是換句話說**——要用不同訴求覆蓋面。

每組依 §3 結構填齊欄位，並標明 `angle / hook_formula / funnel_stage`（逼出差異、方便人工挑）；`composition_prompt` 規格見 §3C。

### 3C. 共用：`composition_prompt` 規格
不論模式，每個 `composition_prompt` 必須具體寫出：
1. 主題 / 主體 / 場景
2. 配色 / 光線 / 氛圍 / 比例（用 `aspect`）
3. **圖中文字（佔位引用）**：標明每段文字的**位置**，文字本身一律寫成 `{{content.欄位}}`（如 `{{content.title}}`、`{{content.cta}}`），實際字串放進該創意的 `content`；**不要把字串寫死在 prompt 裡**。
4. **合規負面表列**：依 `compliance` 明確寫「畫面不得出現…」（若有需求）。

### 4. 自評挑選（衝量也要顧品質）
產完逐項快速自評：鉤子夠不夠強？視覺方向是否和別組重複？合規有沒有踩線？圖中文字會不會太多塞不下？
不及格的就**修或汰換**，確保交付的每組都可用。回報時可附一句各組的自評重點。

### 5. 寫檔
結構與範例見 §3。依情況決定**追加**或**另開檔**：
- **新的一批**（全新 brief）：存成 `data/creatives/<id>.json`（`id` = 時間戳 + 品牌 slug，用 Bash 取）。
- **同一批再加**（「再多幾組 / 換切角」等同一 brief 的延伸）：**讀回該批 `<id>.json`，把新組 append 到 `creatives[]` 後面再寫回**，**不覆蓋、不另開檔**（原有的組保留）。
- 換尺寸或不確定要不要併入同一批時，**先問使用者**。

### 6. 產後
回報：產了幾組、存到哪；模式 B 附（切角 × hook × 漏斗）一覽、模式 A 附各文案的視覺版本數。
主動問是否要：**再多幾組 / 換切角 / 換尺寸(aspect)**（「再多幾組 / 換切角」＝追加到同一批，見 §5）。
產出可在 `serve` 看板的「創意產出」區檢視（也可在看板編輯文案/構圖、生圖、刪除）。
