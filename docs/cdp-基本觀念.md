# CDP 基本觀念（從啟動一台可遙控的 Chrome，到自動化工具接管）

> 給看不懂「為什麼要開一台奇怪的 Chrome」的人。讀完你會懂：那台帶「除錯埠」的 Chrome 是什麼、
> 自動化工具（以 Playwright 為例）怎麼「接管」它、整個流程一步步發生什麼。不需要任何前置知識。
>
> 這份是**純觀念教學、用例子說明**；本專案實際怎麼跑，最後一段帶你過去。

---

## 一句話先講

**CDP（Chrome DevTools Protocol）＝Chrome 內建的一條「遙控頻道」。** 先開一台「打開這條頻道」的 Chrome，
再讓自動化工具（例如 Playwright）透過這條頻道去點按鈕、打字、讀畫面——就像有人在旁邊用遙控器操作那台瀏覽器。

---

## CDP 是什麼？用比喻理解

你每天用的 Chrome，內部一直有一套「**開發者工具**」（按 F12 跳出來那個面板）。那個面板能做的事——
看網頁原始碼、改文字、模擬點擊、抓網路請求——背後靠的就是一條協定，叫 **Chrome DevTools Protocol，簡稱 CDP**。

把它想成 Chrome 機身上的一個 **遙控接收埠**：

- 平常這個埠是**關著的**（沒人能從外面遙控你的瀏覽器，安全）。
- 啟動 Chrome 時若加一句「**把遙控埠打開，設在 9222 號**」，這台 Chrome 就會開一條頻道，
  任何知道「9222」的程式都能連進來、對它下指令。

> 9222 只是個慣用的「門牌號碼」（port），換成別的數字也行，只要兩邊講好同一個號碼。

---

## 兩種玩法：「自己開一台」 vs 「接管現成的」

自動化工具操作瀏覽器有兩種路子，先分清楚：

- **自己開一台全新的**（最常見）：工具啟動一個乾淨的瀏覽器來跑。缺點是那台「空白」——沒有你的登入，
  而且帶著機器人特徵（見下面隱蔽性那段），容易被網站擋。
- **接管一台現成的（CDP）**：你先自己開一台帶除錯埠的真 Chrome、登入好，工具再透過 CDP「**接手**」這台。
  好處是用的是**你本人、已登入**的瀏覽器。

CDP 講的就是第二種，下面都圍著「接管」說。

---

## 它能拿來做什麼？（用途，舉例）

- **接管「已經登入」的網頁去自動化**——不用重寫登入、不碰你的密碼。
  例：接管你已登入的 ChatGPT 網頁自動生圖、或自動操作某個內部後台。
- **讓人或 AI 盯著畫面迭代**——一邊操作、一邊把結果讀回來判斷下一步、不滿意再調。
- **比一般自動化更不易被偵測**——因為那是你本人開的真瀏覽器（後面解釋）。

---

## 登場角色

| 角色 | 是什麼 |
|------|--------|
| **Chrome（帶除錯埠）** | 一台打開了 9222 遙控頻道的真瀏覽器；你在裡面登入要操作的網站 |
| **9222 除錯埠** | 那條遙控頻道的「門牌」：啟動參數 `--remote-debugging-port=9222` |
| **自動化工具（如 Playwright）** | 透過 CDP 連上去、對 Chrome 下指令的程式 |
| **下指令的「大腦」** | 決定「點哪、打什麼、好不好」的那一方——可能是你寫的腳本，也可能是 AI |

一句話串起來：**大腦出主意 → 自動化工具動手 → 透過 CDP 遙控那台 Chrome → Chrome 在網頁上完成操作。**

---

## 從啟動到接管：一步步（以 Playwright 為例）

```
 啟動者               Chrome(9222)          自動化工具(Playwright)      大腦(你的程式 / AI)
   │                     │                         │                        │
 ① 啟動帶除錯埠的 Chrome  │                         │                        │
   ├────────────────────►│ 開一台 Chrome、           │                        │
   │                     │  打開 9222 遙控埠          │                        │
   │                     │                         │                        │
 ② 在這台 Chrome 登入要操作的網站                     │                        │
   ├────────────────────►│ 記住登入態(cookie)        │                        │
   │                     │                         │                        │
   │       ③ 工具用 --cdp-endpoint / connectOverCDP 連上 9222                  │
   │                     │◄────────────────────────┤                        │
   │                     │                         │  「接管成功」            │
   │                     │           ④ 大腦下指令：開分頁、輸入、點擊…           │
   │                     │◄────────────────────────┼────────────────────────┤
   │                     │ 在網頁上執行              │                        │
   │                     │           ⑤ 把畫面/結果讀回來給大腦判斷 ───────────►│ 決定下一步
```

逐步白話：

1. **啟動一台帶除錯埠的 Chrome**：用一句特別的指令把 Chrome 叫起來——
   ```
   chrome --remote-debugging-port=9222 --user-data-dir=<一個專用資料夾>
   ```
   - `--remote-debugging-port=9222`：**打開 9222 這條遙控頻道**（重點就這句）。
   - `--user-data-dir=…`：給這台 Chrome **專屬的設定/登入資料夾**，跟你日常 Chrome 分開、互不干擾；登入態存這（下次免再登）。
   > 這一步「誰來做」都行——你手動跑、雙擊一個腳本、或讓程式自動幫你跑，**效果完全一樣**。

2. **在這台 Chrome 登入你要操作的網站**（例如 chatgpt.com），視窗保持開著。
   登入是「你本人」做的，所以這台瀏覽器跟正常人用的沒兩樣。

3. **自動化工具透過 CDP 接管**：它不另外開瀏覽器，而是去「9222 找一台現成的 Chrome 接手」——
   Playwright 寫法是 `--cdp-endpoint http://127.0.0.1:9222`（程式碼裡叫 `connectOverCDP`）。
   `127.0.0.1` ＝你自己這台電腦，這條頻道只在本機、外面連不到。

4. **接管成功後，大腦就能操作那台 Chrome**：開新分頁、在輸入框打字、點按鈕、送出。

5. **把畫面/結果讀回來**判斷下一步——不滿意就再調、再來一次。

## 為什麼這樣「比較不容易被偵測」？

很多網站會擋「自動化瀏覽器」。一般自動化（讓程式自己開一台全新 Chrome）會留下機器人痕跡——
例如 `navigator.webdriver = true`、特殊啟動參數等，網站一驗就知道。

「接管現成的真 Chrome」不一樣：

- 瀏覽器是**真人手動開、手動登入**的，不是程式憑空生出來的。
- 工具只是**事後從遙控埠接手**，沒有改動它「我是不是機器人」的那些特徵。

所以從網站角度看，它就是「一個正常人在用 Chrome」，自然順很多。

> ⚠️ 這只在「**自己用自己的帳號、吃自己的訂閱**」才合理。別拿去做違反對方服務條款的事。

---

## 卡住了？常見狀況

| 症狀 | 多半原因 | 怎麼辦 |
|------|---------|--------|
| 工具連不到瀏覽器 | 那台帶除錯埠的 Chrome **沒開**、或開了**沒登入** | 重開那台 Chrome、登入、視窗別關 |
| 連線位置不對 | 埠號改過，或工具指到錯的 host:port | 兩邊用同一個位置（預設 `127.0.0.1:9222`） |
| 工具本身起不來 | 缺它的執行環境（如 Playwright 需要 Node） | 把對應環境裝好 |

---

## 名詞小抄

- **CDP（Chrome DevTools Protocol）**：Chrome 內建的遙控協定，DevTools（F12）和 Playwright 都靠它。
- **除錯埠 / remote debugging port（9222）**：那條遙控頻道的門牌號碼。
- **`--remote-debugging-port`**：啟動 Chrome 時打開遙控埠的參數。
- **`--user-data-dir`**：那台 Chrome 的專屬資料夾，存它自己的登入態、設定，跟日常 Chrome 分開。
- **Playwright**：微軟出的瀏覽器自動化工具；可以「自己開一台」，也可以用 CDP「**接管**」現成那台。
- **`--cdp-endpoint` / `connectOverCDP`**：叫工具「**去這個位置接管一台現成的瀏覽器**」，而不是自己開新的。
- **`127.0.0.1`**：指「**你自己這台電腦**」；這條遙控頻道只在本機，外網連不進來。
