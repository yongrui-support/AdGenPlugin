# CDP 基本觀念（從啟動到 Playwright 接管）

> 給看不懂「為什麼要開一台奇怪的 Chrome」的人。讀完你會知道：那台帶除錯埠的 Chrome 是什麼、
> Playwright 怎麼「接管」它、整個流程一步步發生了什麼事。不需要任何前置知識。

---

## 一句話先講

**CDP（Chrome DevTools Protocol）＝Chrome 內建的一條「遙控頻道」。** 我們先開一台「打開這條頻道」的 Chrome，
再讓 Playwright（程式）透過這條頻道去點按鈕、打字、讀畫面——就像有人在旁邊用遙控器操作你的瀏覽器。

---

## CDP 是什麼？用比喻理解

你每天用的 Chrome，其實內部一直有一套「**開發者工具**」（按 F12 會跳出來的那個面板）。那個面板能做的事——
看網頁原始碼、改文字、模擬點擊、抓網路請求——背後靠的就是一條協定，叫 **Chrome DevTools Protocol，簡稱 CDP**。

把它想成 Chrome 機身上的一個 **遙控接收埠**：

- 平常這個埠是**關著的**（沒人能從外面遙控你的瀏覽器，安全）。
- 如果你啟動 Chrome 時加一句「**把遙控埠打開，設在 9222 號**」，這台 Chrome 就會在背景開一條頻道，
  任何知道「9222」的程式都能連進來、對它下指令。

> 9222 只是個慣用的「門牌號碼」（port），換成別的數字也行，只要兩邊講好同一個號碼。

---

## 為什麼這個專案要用它？

生圖如果走 OpenAI **API**：又貴、又慢、還不能讓我（Claude）一張張盯著品質改。

改用 CDP 接管「**你已經登入的 ChatGPT 網頁**」就解決了：

1. **吃訂閱、不另外花錢**——用的是你 ChatGPT Plus 的網頁額度，不是按次計費的 API。
2. **我能用眼睛盯品質**——我直接操作網頁、把生成的圖讀回來看，不滿意就改提示詞重生，迭代到過關。
3. **比較不會被當成機器人**——因為那「真的是一台你本人開、你本人登入的 Chrome」（後面會解釋）。

---

## 登場角色（先認識這四個）

| 角色 | 是什麼 | 在這專案裡 |
|------|--------|-----------|
| **Chrome（帶除錯埠）** | 一台打開了 9222 遙控頻道的真瀏覽器 | 由 `.browser/launch-chrome-cdp` 腳本啟動，你在裡面登入 ChatGPT |
| **9222 除錯埠** | 那條遙控頻道的門牌 | `--remote-debugging-port=9222` |
| **Playwright（＋MCP）** | 一個會「透過 CDP 下指令」的瀏覽器自動化程式 | 內建在本 plugin 的 `.mcp.json`，用 `--cdp-endpoint` 連上 9222 |
| **Claude（我）** | 真正決定「要點哪、打什麼字、圖好不好」的大腦 | 透過 MCP 工具呼叫 Playwright |

一句話串起來：**Claude 出主意 → 叫 Playwright 動手 → Playwright 透過 CDP 遙控那台 Chrome → Chrome 在 ChatGPT 網頁上生圖。**

---

## 從啟動到接管：一步步發生什麼事

```
  你                      Chrome(9222)            Playwright/MCP            Claude(我)
  │                          │                        │                       │
  │ ① 跑啟動腳本             │                        │                       │
  ├─────────────────────────►│ 開一台 Chrome，        │                       │
  │                          │   並打開 9222 遙控埠    │                       │
  │                          │                        │                       │
  │ ② 在這台 Chrome          │                        │                       │
  │    手動登入 chatgpt.com  │                        │                       │
  ├─────────────────────────►│ 記住登入態(cookie)     │                       │
  │                          │                        │                       │
  │ ③ (重)啟動 Claude Code    │                        │                       │
  │                          │                ④ MCP 啟動，用 --cdp-endpoint   │
  │                          │◄───────────────── 連上 9222 ──────────────────┤
  │                          │                        │  「接管成功」          │
  │                          │                        │                       │
  │                          │           ⑤ Claude 下指令：開新分頁、貼提示詞、送出
  │                          │◄───────────────────────┼───────────────────────┤
  │                          │ ChatGPT 生圖            │                       │
  │                          │                ⑥ 把生成圖讀回來給 Claude 看 ──►│ 不滿意→改提示重生
  │                          │                        │                       │ 滿意→存檔
```

逐步白話版：

1. **你跑啟動腳本**（Windows 雙擊 `.bat`／mac·linux 跑 `.sh`）。腳本做的事就是用一句特別的指令把 Chrome 叫起來：
   ```
   chrome.exe --remote-debugging-port=9222 --user-data-dir=<一個專用資料夾>
   ```
   - `--remote-debugging-port=9222`：**打開 9222 這條遙控頻道**（重點就這句）。
   - `--user-data-dir=…`：給這台 Chrome 一個**專屬的設定/登入資料夾**，跟你平常那個 Chrome 分開，互不干擾；
     登入態存在這（下次免再登）。

2. **你在這台 Chrome 裡手動登入 ChatGPT**，然後**視窗保持開著別關**。
   （登入是「你本人」做的，所以這台瀏覽器跟正常人用的沒兩樣。）

3. **（重）啟動 Claude Code**。

4. **Playwright MCP 自動連上 9222**。它不另外開瀏覽器，而是「**接管你那台已經開好的**」——
   設定裡那句 `--cdp-endpoint http://127.0.0.1:9222` 就是在說「去 9222 找一台現成的 Chrome 接手」。
   （`127.0.0.1` ＝你自己的電腦，這條頻道只在本機、外面連不到。）

5. **接管成功後，我（Claude）就能操作那台 Chrome**：開新分頁、在 ChatGPT 輸入框貼提示詞、附素材圖、送出。

6. **生成的圖我會讀回來自己看**，對照水準、不滿意就改提示詞重生，滿意才存檔。

## 為什麼這樣「比較不容易被偵測」？

很多網站會擋「自動化瀏覽器」。一般的自動化（讓程式自己開一台全新 Chrome）會留下機器人痕跡——
例如 `navigator.webdriver = true`、特殊啟動參數等，網站一驗就知道。

我們的做法不一樣：

- 瀏覽器是**你本人手動開、手動登入**的真 Chrome，不是程式憑空生出來的。
- 我們只是**事後從遙控埠接手**，沒有改動它「我是不是機器人」的那些特徵。

所以從網站角度看，它就是「一個正常人在用 Chrome」，自然順很多。

> ⚠️ 這是「**自己用自己的帳號、吃自己的訂閱**」才合理。別拿去做違反對方服務條款的事。

---

## 卡住了？常見三種狀況

| 症狀 | 多半原因 | 怎麼辦 |
|------|---------|--------|
| MCP 工具連不到瀏覽器 | 那台 9222 Chrome **沒開**、或開了**沒登入** | 重跑啟動腳本、登入 ChatGPT、視窗別關，再 resume Claude Code |
| MCP 根本起不來 | 沒裝 **Node/npx**（Playwright MCP 靠它跑） | 裝 Node（見 `setup` skill 第 5 步） |
| 連線位置不對 | 你把埠改成別的號碼 | 設環境變數 `PLAYWRIGHT_CDP_URL` 指向新位置 |

## 名詞小抄

- **CDP（Chrome DevTools Protocol）**：Chrome 內建的遙控協定，DevTools（F12）和 Playwright 都靠它。
- **除錯埠 / remote debugging port（9222）**：那條遙控頻道的門牌號碼。
- **`--user-data-dir`**：給那台 Chrome 的專屬資料夾，存它自己的登入態、設定，跟你日常 Chrome 分開。
- **Playwright**：微軟出的瀏覽器自動化工具；這裡不讓它自己開瀏覽器，而是**接管**現成那台。
- **MCP（Model Context Protocol）**：讓 Claude 能呼叫外部工具（這裡是 Playwright）的橋；本 plugin 已內建設定。
- **`--cdp-endpoint`**：告訴 Playwright「**去這個位置接管一台現成的瀏覽器**」，而不是自己開新的。
- **`127.0.0.1`**：指「**你自己這台電腦**」；這條遙控頻道只在本機，外網連不進來。
